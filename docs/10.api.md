# API設計（案）

## 前提
- データソースはAPI先行設計
- 1日1レコードの上書きはサーバ側で実施（POSTでUpsert）
- 取得系エンドポイントは `/records`（一覧）と `/records/:date`（詳細）
- Supabase を使用予定（このリポジトリでは設定/実装しない）
- 認証/認可は書き込み系のみ必須（POST/PATCH/DELETE）
- プロフィール（体重）はSupabaseに保存する

## エンドポイント
### POST /records
- 用途: 1日1レコードの作成
- 挙動: 同日が存在する場合はエラーを返す（上書きしない）

### GET /records
- 用途: 一覧取得
- 並び順: 日付の新しい順（降順）
- クエリ: `page=1&limit=10`（ページング対応）
- 返却項目（最小）:
  - date
  - memo
  - workouts (part/name/sets/reps/weight)
  - cardios (type/minutes/distance) ※複数行
  - totalSets
  - totalCalories
- ページング情報:
  - totalCount
  - page
  - limit
  - totalPages

### GET /records/:date
- 用途: 詳細取得
- 返却項目（最小）:
  - date
  - memo
  - workouts (part/name/sets/reps/weight)
  - cardios (type/minutes/distance) ※複数行

### GET /records/calendar?month=YYYY-MM
- 用途: カレンダー用の月別記録有無取得
- 返却項目: 記録がある日付の配列

### GET /records/trends?period=1w|1m|3m|all
- 用途: 推移グラフ用データ取得
- 返却項目:
  - dates (日付配列)
  - weight (体重配列)
  - totalSets (セット数配列)
  - cardioDistance (距離配列)
  - calories (カロリー配列)

## 管理者専用API
### 範囲
- 削除 / 編集 / マスター管理 / エクスポート（Issue #20 で削除予定）

### GET /admin/export（Issue #20 で削除予定、現時点では現役）
- 用途: データ出力（管理者のみ）
- クエリ: `from=YYYY-MM-DD`, `to=YYYY-MM-DD`, `format=csv|json`
- 実装: `/api/admin/export`

### DELETE /records/:date
- 用途: レコード削除（管理者のみ）

### PATCH /records/:date
- 用途: レコード編集（管理者のみ）

### マスター管理
- `GET /masters?type=body-parts|exercises|cardio-types`
- `POST /masters?type=...`
- `PATCH /masters/:id`
- `DELETE /masters/:id`

### プロフィール
- `GET /profile`（体重取得）
- `POST /profile`（体重保存）
- 暫定: `/api/profile` はフロントの仮実装で使用（本番はSupabase想定）

## バリデーション方針
- フィールド単位でエラーを返す（フロント・サーバ双方で検証）
- エラーがある場合は保存を抑止する
