# API設計（案）

## 前提
- データソースはAPI先行設計
- 1日1レコードの上書きはサーバ側で実施（POSTでUpsert）
- 取得系エンドポイントは `/records`（一覧）と `/records/:date`（詳細）
- Supabase を使用予定（このリポジトリでは設定/実装しない）
- 認証/認可は書き込み系のみ必須（POST/PATCH/DELETE）
- プロフィール（体重）はSupabaseに保存する

## エンドポイント
### POST /records
- 用途: 1日1レコードの作成
- 挙動: 同日が存在する場合はエラーを返す（上書きしない）

### GET /records
- 用途: 一覧取得
- 並び順: 日付の新しい順（降順）
- 返却項目（最小）:
  - date
  - totalSets
  - cardioMinutes
  - cardioDistance

### GET /records/:date
- 用途: 詳細取得
- 返却項目（最小）:
  - date
  - memo
  - workouts (part/name/sets/reps/weight)
  - cardio (type/minutes/distance)

## 管理者専用API
### 範囲
- 削除 / 編集 / マスター管理 / エクスポート

### DELETE /records/:date
- 用途: レコード削除（管理者のみ）

### PATCH /records/:date
- 用途: レコード編集（管理者のみ）

### GET /admin/export
- 用途: データ出力（管理者のみ）
- クエリ: `from=YYYY-MM-DD`, `to=YYYY-MM-DD`, `format=csv|json`
- 実装: `/api/admin/export`（フロント実装）

### マスター管理
- `GET /masters?type=body-parts|exercises|cardio-types`
- `POST /masters?type=...`
- `PATCH /masters/:id`
- `DELETE /masters/:id`

### プロフィール
- `GET /profile`（体重取得）
- `POST /profile`（体重保存）
- 暫定: `/api/profile` はフロントの仮実装で使用（本番はSupabase想定）
