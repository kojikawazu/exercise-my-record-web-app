# 03 機能仕様書（Functional Specification）

機能詳細・ユーザーフロー・UI/UX 仕様・ビジネスロジックを定義する（確定仕様）。

- 記録追加 / 記録編集（同一フォームを共用）
- 一覧
- 詳細
- カレンダー（月表示）※未実装（設計のみ）
- 推移グラフ ※未実装（設計のみ）
- 管理者メニュー
- プロフィール（体重設定）

## ユーザーフロー（画面遷移）

### 一般ユーザー

- 一覧 → 詳細（「詳細を見る」）
- 詳細 → 一覧（「一覧へ戻る」）
- 一覧 → カレンダー（ナビゲーション）※カレンダーは未実装
- カレンダー → 詳細（日付クリック）※未実装
- 一覧 → 推移グラフ（ナビゲーション）※推移グラフは未実装

### 管理者

- 一覧 → 管理者ログイン（ヘッダー右上リンク）→ 一覧（ログイン成功時）
- 一覧 → 記録追加（一覧上部「記録追加」）→ 一覧（保存後）
- 詳細 → 記録編集（「編集」ボタン）→ 一覧（保存後）
- 管理者メニュー → 記録一覧(管理操作) / 記録追加 / プロフィール / マスター管理 / データ出力（Issue #20 で削除予定）/ ログアウト

### 制御

- 未ログインで管理者 URL にアクセスした場合はログイン画面へリダイレクト。
- 管理者メニューはログイン後のみ表示（一般ユーザー非表示）。
- ログアウト導線は画面右上に固定表示。
- 主な編集導線は一覧/詳細画面から直接行う（管理者メニューは補助的な位置付け）。

## 機能詳細（画面別仕様）

### 一覧

- 並び順は日付の新しい順（降順）、1 日 1 カード表示。
- 表示項目: 日付 / 筋トレメニュー一覧 / 有酸素メニュー一覧 / 推定消費カロリー / 詳細ボタン。
  - 筋トレメニュー一覧: 各行の部位・種目・セット数・回数・重量。
  - 有酸素メニュー一覧: 各行の種別・時間・距離（複数行対応）。
- 空状態文言: 「記録がありません。最初の記録を追加しましょう」。
- 管理者は一覧上部に「記録追加」ボタンを表示。
- ヘッダーはタイトル＋管理者ログインリンク（右上）。
- 詳細ボタン文言は「詳細を見る」。
- 推定消費カロリーをカード下部に表示。
- ページング: 1 ページ 10 件、前へ/次へボタンで切替。

### 詳細

- 並び順: 日付 → 筋トレ一覧 → 有酸素一覧 → 体調メモ。
- 筋トレ一覧はカード/リスト形式（表示順: 種目 → 部位 → セット → 回数 → 重量、ラベル付き・単位付き）。
- 有酸素一覧は複数行表示（種別/時間/距離をラベル付き・単位付きで行ごとに表示）。
- 体調メモが空の場合は「体調メモなし」を表示。
- ヘッダーはタイトル＋戻るリンク（「一覧へ戻る」）。
- 推定消費カロリーを上部に表示。
- 管理者向けに「編集」ボタンを表示（クリックで記録編集画面へ遷移）。

### 記録追加 / 記録編集

- 記録追加と記録編集は同一フォームを共用（編集時は既存データをプリセット）。
- 日付初期値は空（ユーザーが選択）。編集時は既存の日付をセットし変更不可。
- 筋トレ行は初期表示で 1 行、最少 1 行を維持（全削除不可）。
- 有酸素行は初期表示 0 行（追加ボタンで追加）、全削除可能（任意項目のため）。
- 体調メモの入力 UI はテキストエリア。
- 保存後は一覧へ戻る。
- ヘッダーはタイトル＋戻るリンク（「一覧へ戻る」）。
- 筋トレ/有酸素行の追加ボタンはセクション末尾、削除ボタンは各行の右端に配置。
- 入力ラベルを表示。
- フィールド単位のバリデーションエラーを各入力の下に赤文字で表示（保存ボタン押下後から表示）。エラーがある場合は保存を抑止。
- 日付ピッカーはモダンなカレンダー UI を使用し、年へのジャンプ操作を用意。
- 推定消費カロリーを画面下部に表示。

### カレンダー ※未実装（設計のみ。[`11-tasks.md`](./11-tasks.md) で未着手）

- 月表示のカレンダーを表示。
- 記録がある日にマーク（ドット/ハイライト）を表示。
- 日付クリックで詳細画面へ遷移。
- 前月/次月のナビゲーションボタン。ヘッダーはタイトル＋ナビゲーション。

### 推移グラフ ※未実装（設計のみ。[`11-tasks.md`](./11-tasks.md) で未着手）

- 折れ線グラフで記録の推移を表示。
- 表示指標: 体重 / 合計セット数 / 有酸素距離 / 推定消費カロリー。
- 期間選択: 1 週間 / 1 ヶ月 / 3 ヶ月 / 全期間。ヘッダーはタイトル＋期間選択。

### 管理者メニュー

- ログイン後のみ表示（一般ユーザー非表示）、カード/ボタンのグリッド。
- 並び順: 記録一覧(管理操作) → 記録追加 → マスター管理 → データ出力（Issue #20 で削除予定）→ ログアウト。
- カード文言は短い名詞で統一。ログアウト導線は画面右上に固定表示。

### マスター管理

- 「部位」「種目」「有酸素種別」のタブ切替。初期値を用意し管理者が編集可能。
- 一覧はリスト表示（名称 + 編集/削除アイコン）。追加は一覧上部の入力＋追加ボタン、編集はインライン編集、削除は確認ダイアログ。
- 追加入力のプレースホルダー「新しい項目を追加」、削除確認文言「この項目を削除します。よろしいですか？」。

### プロフィール

- 体重（kg）を入力・保存。データは 1 件のみ維持（上書き）。

### データ出力（Issue #20 で削除予定、現時点では現役）

- CSV/JSON 両対応。期間指定は From/To の日付ピッカー。出力形式はラジオボタン（CSV/JSON）。出力ボタン文言は「エクスポート」。

## ビジネスロジック

### 入力 / バリデーション

- 数値項目: セット数、回数、重量、時間、距離。小数入力を許可。
- 有酸素種別は「ラン/ウォーク」。
- 1 日に複数の筋トレ項目・複数の有酸素項目を追加可能。
- バリデーションはフィールド単位でエラー表示（どの項目がエラーか分かるようにする）。
- エラー表示は保存ボタン押下後から開始（初期表示ではエラーを出さない）。エラーがある場合は保存を抑止。
- 同日保存時はエラーとして弾き、通知 UI を表示する。
- 実装: `front/src/lib/validation.ts`（純粋関数）、`front/src/hooks/useRecordValidation.ts`（状態管理）。

### 単位

- 重量 = kg / 距離 = km / 時間 = 分。

### 推定消費カロリー（目安）

目安（概算）で算出する。体重はプロフィール（Supabase）に保存した値を使用。一覧/詳細/記録追加の全てに「推定消費カロリー」ラベルで表示する。

#### 有酸素（時間ベース）

- `消費カロリー = METs × 体重(kg) × 時間(時間)`
- METs は種別ごとの固定値（暫定）: ラン = 8.0 / ウォーク = 4.0。
- 未知の種別は MET = 0（カロリー 0）。

#### 筋トレ（概算）

- `消費カロリー = 体重(kg) × 0.1 × セット数合計`（セット数合計はその日の合計）。

#### 表示

- 一覧はカード下部、詳細は上部、記録追加は画面下部に表示。
- 整数に丸めて「N kcal」形式で表示（実装: `front/src/lib/calorie.ts`）。
- 未決定: METs 係数の調整。
