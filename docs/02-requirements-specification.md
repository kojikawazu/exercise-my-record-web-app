# 02 要件仕様書（Requirements Specification）

機能要件一覧・受け入れ基準・優先度を定義する。画面・UI の詳細仕様は [`03-functional-specification.md`](./03-functional-specification.md) を参照。

## ロール・権限

| ロール | 権限 |
|--------|------|
| 管理者 | 記録の追加/編集/削除、マスター管理、プロフィール（体重）、データ出力（Issue #20 で削除予定） |
| 一般ユーザー | 一覧/詳細/カレンダー/推移グラフの閲覧のみ（ログイン不要） |

## 機能要件一覧

| ID | 機能 | 概要 | ロール |
|----|------|------|--------|
| FR-001 | 記録追加 | 1 日 1 レコードを作成（同日保存はエラー） | 管理者 |
| FR-002 | 記録編集 | 既存レコードを編集（追加と同一フォームを共用） | 管理者 |
| FR-003 | 記録削除 | 既存レコードを削除 | 管理者 |
| FR-004 | 一覧表示 | 日付降順・1 日 1 カード・ページング（10 件/ページ） | 全ユーザー |
| FR-005 | 詳細表示 | 日付/筋トレ/有酸素/体調メモ/推定消費カロリーを表示 | 全ユーザー |
| FR-006 | カレンダー表示 | 月表示。記録がある日にマーク、日付クリックで詳細へ | 全ユーザー |
| FR-007 | 推移グラフ表示 | 体重/合計セット数/有酸素距離/推定消費カロリーの折れ線、期間選択 | 全ユーザー |
| FR-008 | 推定消費カロリー算定 | 一覧/詳細/記録追加に表示（算定式は 03 参照） | 全ユーザー |
| FR-009 | バリデーション | フィールド単位でエラー表示、保存抑止（保存押下後から） | 管理者 |
| FR-010 | 認証 | 管理者は Google OAuth、未ログインで管理者 URL は `/admin/login` へ | 管理者 |
| FR-011 | マスター管理 | 部位/種目/有酸素種別の CRUD | 管理者 |
| FR-012 | プロフィール | 体重（kg）の保存（1 件維持・上書き） | 管理者 |
| FR-013 | データ出力 | CSV/JSON エクスポート（期間指定）※ Issue #20 で削除予定 | 管理者 |

## データ要件

### 筋トレ（必須）

- 部位（セレクト）/ 種目名（テキスト）/ セット数（数値）/ 回数（数値）/ 重量（数値・kg）。

### 有酸素（任意・複数行）

- 種別（ラン/ウォーク）/ 時間（数値・分）/ 距離（数値・km）。

### 任意

- 体調メモ（500 文字まで、改行可）。

## 必須画面

1. 記録追加 / 記録編集
2. 一覧
3. 詳細
4. カレンダー
5. 推移グラフ
6. 管理者メニュー（プロフィール・マスター管理・データ出力を含む）

## 必須 UI 部品

- 日付ピッカー（カレンダー UI + 年ジャンプ）、数値入力、セレクト（部位・ラン/ウォーク）、筋トレ/有酸素行の追加/削除、バリデーション表示、保存ボタン、空状態表示、ページング。

## 受け入れ基準

主要な受け入れシナリオは E2E ケースとして [`08-test-specification.md`](./08-test-specification.md) に定義する（一覧/詳細表示、ログイン/メニュー出し分け、記録 追加/編集/削除、マスター管理、データ出力 等）。

## 優先度

実装済み機能は [`11-tasks.md`](./11-tasks.md) を参照。デザイン改善の優先度（全員合意）は以下（詳細は [`docs/design-discussion/02-design-recommendations.md`](./design-discussion/02-design-recommendations.md)）。

| 優先度 | 項目 |
|--------|------|
| P0 | ダークモード対応 / テンプレート・前回コピー |
| P1 | トップページのサマリー化 / ストリーク表示 / 配色リブランディング |
| P2 | 自己ベスト通知・前回比較 / ヒートマップカレンダー / ステッパー / 種目マスター選択化 |
| P3 | 種目別重量推移グラフ / 月間サマリー自動生成 / モバイルファースト改修 |
