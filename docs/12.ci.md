# CI/CD パイプライン

## 概要

GitHub Actions による自動テストパイプライン。
PR 作成時および `main` ブランチへの push 時に Vitest ユニットテストと Playwright E2E テストを実行する。

## ワークフロー構成

ファイル: `.github/workflows/test.yml`

### トリガー

| イベント | 条件 |
|---------|------|
| `push` → `main` | `docs/**` 以外の変更 |
| `pull_request` | `docs/**` 以外の変更 |

### ジョブ

| ジョブ名 | 内容 | 所要時間目安 |
|---------|------|------------|
| `unit-test` | Vitest ユニットテスト（77テスト） | ~30秒 |
| `e2e-test` | Playwright E2E テスト（20テスト） | ~7分 |

2ジョブは並列実行される。

## E2E テスト環境の仕組み

### DB なしで動作する設計

E2E テストは実データベースを使わない。以下の仕組みで DB 依存を排除している。

1. **API モック**: Playwright の `page.route()` でブラウザレベルのネットワークリクエストを傍受
2. **認証バイパス**: `localStorage` に `e2e_admin_bypass=1` を注入して管理者セッションをシミュレート
3. **Supabase ダミー認証情報**: `.env.local` にダミー URL/キーを設定してクライアント初期化を通す

### CI 固有の設定

CI 環境では以下の追加設定が必要:

#### 1. Prisma クライアント生成

```bash
pnpm exec prisma generate
```

`next dev` が API ルートをオンデマンドコンパイルする際に `@/generated/prisma/client` が必要。
未生成だと Module not found エラーが発生し、Next.js Dev Overlay が画面を覆ってテスト操作をブロックする。

#### 2. `.env.local` の動的生成

```bash
echo "NEXT_PUBLIC_E2E_BYPASS=1" >> .env.local
echo "NEXT_PUBLIC_SUPABASE_URL=https://dummy.supabase.co" >> .env.local
echo "NEXT_PUBLIC_SUPABASE_ANON_KEY=dummy-anon-key" >> .env.local
echo "DATABASE_URL=postgresql://dummy:dummy@localhost:5432/dummy" >> .env.local
```

| 変数 | 目的 |
|-----|------|
| `NEXT_PUBLIC_E2E_BYPASS` | 管理者認証バイパスを有効化 |
| `NEXT_PUBLIC_SUPABASE_URL` | Supabase クライアント初期化（ダミー） |
| `NEXT_PUBLIC_SUPABASE_ANON_KEY` | Supabase クライアント初期化（ダミー） |
| `DATABASE_URL` | Prisma 接続文字列（ダミー、実際には接続しない） |

#### 3. Playwright タイムアウト

`playwright.config.ts` で CI 環境のタイムアウトを延長:

- テストタイムアウト: 60秒（ローカル: 30秒）
- webServer 起動タイムアウト: 120秒

`next dev` はページをオンデマンドコンパイルするため、CI（低スペック）では初回アクセスに時間がかかる。

## CI 構築時のトラブルシューティング記録

### Next.js Dev Overlay によるクリックブロック

**症状**: `locator.click: Test timeout exceeded` — ボタンは visible だがクリックできない

**原因**: `prisma generate` 未実行 → `@/generated/prisma/client` の Module not found → Next.js Dev Overlay（`<nextjs-portal>`）が画面全体を覆ってポインターイベントを傍受

**対策**: CI に `prisma generate` ステップを追加

### Supabase クライアント初期化エラー

**症状**: `Error: supabaseUrl is required.` でサーバー起動失敗

**原因**: `NEXT_PUBLIC_SUPABASE_URL` 未設定 → `createClient('', '')` が例外

**対策**: `.env.local` にダミー URL を設定

### NEXT_PUBLIC 変数のクライアントバンドル展開

**症状**: `isBypassAllowed` がクライアントで常に `false`

**原因**: `NEXT_PUBLIC_*` 変数はクライアントバンドルにコンパイル時展開される。シェル環境変数の継承では不十分な場合がある

**対策**: `.env.local` ファイルで確実に設定 + `isBypassAllowed` を `NODE_ENV !== 'production'` のみに簡素化
