## エグゼクティブサマリー

フィットネス愛好家、プロダクトマネージャー、UXリサーチャーの3者によるディスカッションの結果、現在のアプリは「記録帳」としての基本機能は備えているが、**運動者が毎日使いたくなる「トレーニングの相棒」には至っていない**という共通認識が得られた。

以下の4つの改善軸が全員一致で挙がった:
1. **ダークモード対応** — ジム環境での視認性
2. **入力の省力化** — テンプレート/前回コピーによる操作量の劇的削減
3. **進歩の可視化** — サマリー/ストリーク/自己ベスト通知で「成長が見える」体験
4. **配色のリブランディング** — フィットネスアプリとしてのトーン調整

## 1. 配色・トーンの提案

### 現状の課題
- ライトモードのみで、ジムの暗い照明下で白背景が眩しい
- 紫が「瞑想・リラクゼーション」の連想を持ち、筋トレアプリとしてのトーンに合わない可能性
- 背景とカードのコントラストが弱い（`#f9fafb` vs `#ffffff`）

### 提案: ダーク＆エナジーカラースキーム

#### ダークモード（メイン・デフォルト）
| 要素 | カラー | 用途 |
|------|--------|------|
| 背景 | `#0f0f14`〜`#1a1a2e` | ページ背景（深いダーク） |
| サーフェス | `#1e1e30`〜`#252540` | カード・パネル |
| プライマリ | `#6c5ce7`〜`#7c3aed` | ブランドカラー（紫を活かしつつモダンに） |
| アクセント | `#00d2a0`〜`#10b981` | 達成・成長・ポジティブ指標（グリーン系） |
| CTA | `#f97316`〜`#ff6b35` | 主要アクション（オレンジ＝エナジー） |
| デストラクティブ | `#ef4444` | 削除・エラー |
| テキスト | `#f1f5f9` | 通常テキスト |
| サブテキスト | `#94a3b8` | 補助テキスト |

#### ライトモード（サブ・家での振り返り用）
| 要素 | カラー | 用途 |
|------|--------|------|
| 背景 | `#f8f7f4` | ウォームグレー（クリニカルすぎない） |
| サーフェス | `#ffffff` | カード |
| プライマリ | `#6c5ce7` | ブランドカラー |
| アクセント | `#059669` | 達成・成長 |

### 配色の意図
- **紫（プライマリ）**: 競合との差別化を維持。ただし使用範囲を限定（ヘッダー、アクセント）
- **グリーン（アクセント）**: 「健康」「成長」「達成」の意味。自己ベスト更新やポジティブ指標に
- **オレンジ（CTA）**: 行動を促すエナジーカラー。記録追加・保存などの主要アクション

## 2. トップページの再設計

### 現状の課題
- 記録の日付一覧が最初に表示される → 「帳簿」感
- 毎日開く理由がない

### 提案: ダッシュボード型トップ

```
┌─────────────────────────────────────┐
│  今週のサマリー                       │
│  ┌───┐ ┌───┐ ┌───┐ ┌───┐           │
│  │ 3 │ │12 │ │850│ │ 7 │           │
│  │回 │ │set│ │kcal│ │日 │           │
│  │今週│ │合計│ │消費│ │連続│          │
│  └───┘ └───┘ └───┘ └───┘           │
│  先週比: +15% ↑                      │
└─────────────────────────────────────┘
┌─────────────────────────────────────┐
│  ヒートマップカレンダー（今月）         │
│  月 火 水 木 金 土 日                  │
│  ■  □  ■  ■  □  ■  □               │
│  ■  □  ■  ■  ■  □  □               │
│  ■  □  ■  □  ...                    │
└─────────────────────────────────────┘
┌─────────────────────────────────────┐
│  最新の記録  2026-03-21              │
│  ベンチプレス / インクラインDB / ...   │
│  → 詳細を見る                        │
└─────────────────────────────────────┘
```

### 情報の優先順位（全員合意）
1. **進歩の実感** — 今週のサマリー（回数・セット・カロリー・ストリーク）
2. **継続の可視化** — ヒートマップカレンダー
3. **直近のパフォーマンス** — 最新の記録ハイライト
4. **過去の記録一覧** — スクロールまたはタブで表示

## 3. モチベーション要素

### 実装すべき要素（優先順位順）

#### 3-1. ストリーク（連続記録）
- トップページに「○日連続記録中」を目立つ位置に表示
- 連続記録が途切れそうな時に通知（損失回避バイアスの活用）
- カレンダーで連続日をハイライト

#### 3-2. 自己ベスト通知（PR: Personal Record）
- 記録保存時に前回の同種目と比較
- 重量/回数/距離で自己ベスト更新時にアニメーション表示
- 例: 王冠アイコン + 「ベンチプレス 新記録! 100kg」

#### 3-3. 前回比較の常時表示
- 記録詳細に「前回同種目の記録」を薄いグレーで併記
- 例: 「ベンチプレス 100kg（前回: 95kg ↑+5kg）」

#### 3-4. 月間サマリーの自動生成
- 「今月の合計消費カロリー: 12,450 kcal」「先月比 +15%」
- 月間トレーニングボリューム（トータルトン数）

#### 見送る要素
- **バッジ**: ガチ層には子供扱い感。初心者向けとしてのみ将来検討
- **ソーシャル機能**: 個人利用前提のため現時点では不要

## 4. 入力の省力化

### 現状の課題
- 筋トレ1行で5フィールド入力 → インターバル60秒で厳しい
- 種目名がテキスト入力 → 汗で濡れた指でフリック入力は非現実的
- 毎回ゼロから入力 → 曜日ごとにメニューはほぼ同じなのに

### 提案

#### 4-1. テンプレート / 前回コピー（最優先）
- 「前回の記録をコピー」ボタンで筋トレ/有酸素行をプリセット
- 将来的にはルーティン（テンプレート）保存機能

#### 4-2. 種目名のマスター選択化
- テキスト入力 → マスターからのセレクト（既存のマスター管理を活用）
- 部位に「肩」「腕」を追加（最低限: 胸・背中・肩・腕・脚・腹）

#### 4-3. ステッパー（+/-ボタン）
- セット数: +1/-1（デフォルト3）
- 回数: +1/-1（デフォルト10）
- 重量: +2.5kg/-2.5kg
- フリック入力を排除し、タップ操作のみで完結

#### 4-4. モバイルファーストのレイアウト
- 6列グリッド → 2列 or スタック型
- タップターゲット最低44px
- 大きめのボタン、十分なスペーシング

## 5. カレンダー・推移グラフの設計指針

### カレンダー
- **ヒートマップ形式**: 記録あり=色付き、なし=グレー。強度で色の濃淡
- トップページのダッシュボードに小さく組み込み + 専用ページで拡大表示
- 日付タップで詳細へ遷移

### 推移グラフ
- **デフォルト表示期間: 直近4週間**（上昇トレンドが見えやすい期間）
- 1年表示は停滞期が目立ちモチベ低下のリスクがあるため、ユーザーが選択した時のみ
- **種目別重量推移**が最も需要が高い（トレーニーの声）
- 指標: 体重 / 合計セット数 / 有酸素距離 / 推定消費カロリー / 種目別重量

## 6. 将来的な検討事項（今回のスコープ外）

以下はディスカッションで挙がったが、現時点ではスコープ外とする:

| 項目 | 理由 |
|------|------|
| インターバルタイマー | 機能規模が大きい。独立したissueとして検討 |
| セッション（入退館時刻） | データモデルの大幅変更が必要 |
| トレーニングボリューム（トン数計算） | 推移グラフ実装後に追加機能として |
| オンボーディングフロー | デザインリブランディング後に設計 |
| プロダクト名の見直し | ブランディング全体の議論が必要 |
