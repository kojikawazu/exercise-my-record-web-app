# 仕様（確定）

## 目的
フィットネス記録MVPの仕様を明文化し、実装判断を固定する。

## スコープ
- 1日1レコード（同日保存はエラーで弾く）
- 1日に複数の筋トレ種目を追加可能
- 1日に複数の有酸素行を追加可能（任意）

## ロール/権限
- 管理者: 記録の追加/編集/削除、マスター管理、データ出力（Issue #20 で削除予定）
- 一般ユーザー: 一覧/詳細/カレンダー/推移グラフの閲覧のみ

## 認証/セッション
- 認証はGoogle OAuthをMVPに含める
- 管理者ログイン導線は一覧画面に表示する
- 未ログインで管理者URLにアクセスした場合はログイン画面へリダイレクト
- ログイン後は一覧画面へ遷移する
- セッションはブラウザを閉じても保持する

## 画面
- 記録追加
- 記録編集
- 一覧
- 詳細
- カレンダー（月表示）
- 推移グラフ
- 管理者メニュー
- プロフィール（体重設定）
- マスター管理
- データ出力（Issue #20 で削除予定）

## データ仕様
### 筋トレ
- 部位（セレクト）
- 種目名（テキスト）
- セット数（数値）
- 回数（数値）
- 重量（数値）

### 有酸素（複数行追加可）
- 種別（ラン/ウォーク）
- 時間（数値）
- 距離（数値）

### 任意
- 体調メモ（500文字まで、改行可）

## 入力/バリデーション
- 数値項目: セット数、回数、重量、時間、距離
- 数値項目は小数入力を許可する
- 有酸素種別は「ラン/ウォーク」
- 1日に複数の筋トレ項目を追加可能
- 1日に複数の有酸素項目を追加可能（ラン＋ウォーク等）
- バリデーションはフィールド単位でエラーを表示する（どの項目がエラーか分かるようにする）
- エラー表示は保存ボタン押下後から開始する（初期表示ではエラーを出さない）
- 保存ボタン押下後、エラーがある場合は保存を抑止する
- 同日保存時はエラーとして弾き、通知UIを表示する

## 単位
- 重量=kg
- 距離=km
- 時間=分

## 一覧
- 並び順は日付の新しい順（降順）
- 1日1カード表示
- 表示項目: 日付/筋トレメニュー一覧/有酸素メニュー一覧/推定消費カロリー/詳細ボタン
- 筋トレメニュー一覧: 各行の部位・種目・セット数・回数・重量を表示する
- 有酸素メニュー一覧: 各行の種別・時間・距離を表示する（複数行対応）
- 空状態文言: 「記録がありません。最初の記録を追加しましょう」
- 管理者は一覧上部に「記録追加」ボタンを表示する
- ヘッダーはタイトル＋管理者ログインリンク
- 一覧の詳細ボタン文言は「詳細を見る」
- 管理者ログインリンクはヘッダー右上に表示する
- 消費カロリー（目安）をカード下部に表示する
- ページング: 1ページ10件、前へ/次へボタンで切替

## 詳細
- 並び順: 日付→筋トレ一覧→有酸素一覧→体調メモ
- 筋トレ一覧はカード/リスト形式で表示する
- 有酸素一覧は複数行表示（種別/時間/距離を行ごとに表示）
- 体調メモが空の場合は「体調メモなし」を表示する
- ヘッダーはタイトル＋戻るリンク
- 戻るリンク文言は「一覧へ戻る」
- 筋トレ一覧の表示順は種目→部位→セット→回数→重量
- 筋トレ一覧はラベル付きで表示する
- 有酸素一覧はラベル付きで表示する
- 詳細の数値表示は単位付きで表示する
- 消費カロリー（目安）を上部に表示する
- 管理者向けに「編集」ボタンを表示する（クリックで記録編集画面へ遷移）

## 記録追加 / 記録編集
- 記録追加と記録編集は同一フォームを共用する（編集時は既存データをプリセット）
- 日付初期値は空（ユーザーが選択）。編集時は既存の日付をセットし変更不可
- 筋トレ行は初期表示で1行用意する
- 筋トレ行は最少1行を維持する（全削除不可）
- 有酸素行は初期表示で0行（追加ボタンで行を追加する）
- 有酸素行は全削除可能（任意項目のため）
- 体調メモの入力UIはテキストエリア
- 保存後は一覧へ戻る
- ヘッダーはタイトル＋戻るリンク
- 戻るリンク文言は「一覧へ戻る」
- 筋トレ行の追加ボタンはセクション末尾に配置する
- 筋トレ行の削除ボタンは各行の右端に配置する
- 有酸素行の追加ボタンはセクション末尾に配置する
- 有酸素行の削除ボタンは各行の右端に配置する
- 入力ラベルを表示する
- フィールド単位のバリデーションエラーを各入力の下に赤文字で表示する（保存ボタン押下後から表示）
- 保存ボタン押下後、バリデーションエラーがある場合は保存を抑止する
- 日付ピッカーはカレンダーUIを使用し、年へのジャンプ操作を用意する
- 日付ピッカーはモダンなUIコンポーネントを採用する
- 消費カロリー（目安）を画面下部に表示する

## カレンダー
- 月表示のカレンダーを表示する
- 記録がある日にマーク（ドットやハイライト）を表示する
- 日付クリックで詳細画面へ遷移する
- 前月/次月のナビゲーションボタン
- ヘッダーはタイトル＋ナビゲーション

## 推移グラフ
- 折れ線グラフで記録の推移を表示する
- 表示指標: 体重・合計セット数・有酸素距離・推定消費カロリー
- 期間選択: 1週間/1ヶ月/3ヶ月/全期間
- ヘッダーはタイトル＋期間選択

## 管理者メニュー
- 管理者はログイン後に管理者メニューを表示（一般ユーザーには非表示）
- 表示形式はカード/ボタンのグリッド
- 並び順: 記録一覧(管理操作)→記録追加→マスター管理→データ出力（Issue #20 で削除予定）→ログアウト
- ログアウト導線は画面右上に固定表示する
- 管理者メニューのカード文言は短い名詞で統一する
- 主な編集導線は一覧/詳細画面から直接行う（管理者メニューは補助的な位置付け）

## マスター管理
- 「部位」「種目」「有酸素種別」のタブ切替
- 初期値を用意し、管理者が編集可能
- 有酸素種別は管理者が編集可能（追加/削除/名称変更）
- 一覧はリスト表示（名称 + 編集/削除アイコン）
- 追加は一覧上部の入力＋追加ボタン
- 編集はインライン編集
- 削除は確認ダイアログを表示する
- タブのラベルは「部位」「種目」「有酸素種別」
- 追加入力のプレースホルダーは「新しい項目を追加」
- 削除確認ダイアログ文言は「この項目を削除します。よろしいですか？」

## データ出力（Issue #20 で削除予定、現時点では現役）
- CSV/JSON両方対応
- 期間指定はFrom/Toの日付ピッカー
- 出力形式はラジオボタン（CSV/JSON）で選択
- 出力ボタンの文言は「エクスポート」
